# shared-banner

A single JavaScript file, hosted on GitHub Pages, that injects a configurable site banner into any website. Update it once, and every site that includes it updates automatically.

---

## How It Works

Each site loads `banner.js` from this repository via a script tag. Before that script loads, the site optionally defines a `window.BANNER_CONFIG` object with site-specific settings. The banner script reads those settings, falls back to defaults for anything missing, builds a banner element, and injects it into the page at either the top or bottom.

No build step. No framework dependency. No redeployment of individual sites required when the banner changes.

---

## Installation

### Step 1: Add the config block to your site

In your site's `index.html`, paste the following before the closing `</body>` tag. Adjust values to match your site:

```html
<script>
  window.BANNER_CONFIG = {
    siteName: "Your Site Name",
    lastUpdateDate: "2026-03-15",
    lastUpdateShow: true,
    linkHome: true,
    linkApps: true,
    position: "footer",
    sticky: true
  };
</script>
<script src="https://themann00.github.io/shared-banner/banner.js"></script>
```

### Step 2: That is it

The banner will appear on page load using your config values. Any future changes to the banner's design or behavior are made only in this repository.

---

## Configuration Options

All fields are optional. Any field you omit will use the default value listed below.

| Key | Type | Default | Description |
|---|---|---|---|
| `siteName` | string | Text of the first `h1` on the page | Display name shown in the banner |
| `lastUpdateDate` | string (YYYY-MM-DD) | `""` | Date to display as the last update |
| `lastUpdateShow` | boolean | `false` | Whether to show the last update date |
| `linkHome` | boolean | `true` | Whether to show a link to themann00.com |
| `linkApps` | boolean | `true` | Whether to show a link to apps.themann00.com |
| `position` | `"header"` or `"footer"` | `"footer"` | Where the banner appears on the page |
| `sticky` | boolean | `true` | Whether the banner sticks to the top or bottom while scrolling |

### Omitting the config entirely

If a site does not define `window.BANNER_CONFIG` at all, the banner will still load using all defaults. The site name will be pulled from the page's first `h1` element, or fall back to "This Site" if none is found.

---

## Updating the Banner

Make changes to `banner.js` in this repository and push to the main branch. GitHub Pages will serve the updated file within a few minutes. All sites that include the script tag will pick up the changes automatically on their next page load.

No changes to individual sites are needed unless you want to update that site's config values.

---

## Notes

- The script is wrapped in an immediately invoked function expression (IIFE) and does not pollute the global scope beyond reading `window.BANNER_CONFIG`.
- If a site has a strict Content Security Policy that blocks external scripts, the banner will not load. Most static sites generated by Lovable or similar tools do not set restrictive CSPs, but check your browser console for errors if the banner does not appear.
- The sticky banner adds automatic padding to `document.body` to prevent it from overlapping page content.
